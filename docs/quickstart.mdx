# Quickstart

OpenQA scaffolds a `.openqa/` **agent harness** directory inside your existing project. Your feature files live where they always have — the harness wires Claude Code to your Playwright browser.

<Steps>
  <Step title="Run the init wizard">
    ```bash
    npx openqa init
    ```
    The interactive wizard will ask you a few questions.
  </Step>
  <Step title="Choose your agent & model">
    Select **Claude Code** as the agent, then pick a model:
    - `claude-haiku-4-5` — fast, cost-efficient (default)
    - `claude-sonnet-4-6` — balanced
    - `claude-opus-4-7` — most capable
    - Custom model name
  </Step>
  <Step title="Choose your framework">
    - **Playwright-BDD** — Gherkin `.feature` files with Playwright test runner
    - **Cucumber.js** — Standard Cucumber with Playwright browser
  </Step>
  <Step title="Set the path to your feature files">
    Enter the relative path from your project root to your `.feature` files (e.g. `features/` or `tests/features/`). The harness will be configured to point at them automatically.
  </Step>
  <Step title="Authenticate">
    <Tabs>
      <Tab title="Claude CLI (Recommended — no API key needed)">
        ```bash
        claude login
        ```
      </Tab>
      <Tab title="API Key">
        ```bash
        # Add to .openqa/.env
        ANTHROPIC_API_KEY=your_key
        ```
      </Tab>
      <Tab title="OAuth Token (CI)">
        ```bash
        CLAUDE_CODE_OAUTH_TOKEN=your_token
        ```
      </Tab>
    </Tabs>
  </Step>
  <Step title="Write your feature file">
    Use the `*` (asterisk) syntax for a natural, AI-friendly style:

    ```gherkin
    Feature: TodoMVC

      Scenario: Add todo item
        * I navigate to "https://demo.playwright.dev/todomvc/"
        * I add a new todo item "Buy groceries"
        * I should see "Buy groceries" in the todo list
    ```

    Standard `Given`/`When`/`Then` also works.
  </Step>
  <Step title="Run">
    ```bash
    cd .openqa
    npm run test:headed
    ```
  </Step>
</Steps>

## Existing Project?

If you already have feature files, just run `npx openqa init` from your project root, point it at your existing `features/` directory, and the harness will be scaffolded around them.
