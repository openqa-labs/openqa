# OpenQA

**OpenQA is an agent harness for browser test automation.** Write your tests in plain English. A Claude Code agent drives the real browser and executes every step autonomously.

<Note>
  An **agent harness** (as described by [Martin Fowler](https://martinfowler.com/articles/harness-engineering.html) and [Anthropic](https://www.anthropic.com/engineering/managed-agents)) is the infrastructure around an AI model — the scaffolding, tool connections, and control loops that allow an agent to operate reliably within your system. OpenQA is a harness built specifically for browser testing.
</Note>

## How It Works

OpenQA implements the **agent harness** pattern for testing:

1. Your BDD step or YAML test calls `runAgent(claudeCode('model'), 'natural language step', page)`
2. The harness spawns a **Claude Code CLI** subprocess (your AI brain)
3. Claude receives a live connection to **your Playwright browser** via `@playwright/mcp`
4. Claude reads the page, clicks, types, and verifies — using real browser tools
5. The step passes or fails based on what Claude reports back

```typescript
import { test } from "@playwright/test";
import { runAgent, claudeCode } from "openqa";

test("AI fills form", async ({ page }) => {
  await page.goto("https://example.com/form");
  await runAgent(claudeCode('claude-haiku-4-5'), "Fill the contact form with test data", page);
});
```
