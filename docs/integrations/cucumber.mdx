# Cucumber.js Integration

Use the `openqa init` wizard to scaffold a Cucumber.js harness:

```bash
npx openqa init
# Select: Cucumber.js
# Enter the path to your feature files
```

This creates a `.openqa/` directory with everything pre-configured — no step definitions to write.

## Write Feature Files

Use the `*` (asterisk) syntax for natural, AI-friendly steps:

```gherkin
Feature: Shopping Cart

  Scenario: Add product to cart
    * I navigate to "https://shop.example.com"
    * I search for "laptop"
    * I add the first result to cart
    * I should see "1 item" in the cart icon
```

Standard `Given`/`When`/`Then` also works identically.

## Run Tests

```bash
cd .openqa
npm run test:headed
```

See the [Quickstart](/quickstart) for authentication setup.
