# Playwright-BDD Integration

Use OpenQA as the **agent harness** for your Playwright-BDD project. Write feature files in plain English — the harness connects Claude Code to your Playwright browser and executes every step automatically.

## Recommended: Use `npx openqa init`

The easiest way to get started is the interactive wizard, which scaffolds a complete `.openqa/` harness into your existing project:

```bash
npx openqa init
# Select: Playwright-BDD
# Enter the path to your feature files
```

This creates:
- `.openqa/playwright.config.ts` — pre-configured for your feature file path
- `.openqa/steps/fixtures.ts` — Playwright-BDD fixture extension
- `.openqa/steps/steps.ts` — a single AI step definition wired to the harness

## Manual Setup

If you prefer to set up manually:

### Install

```bash
npm install openqa playwright-bdd @playwright/test
```

### Create `fixtures.ts`

```typescript
// steps/fixtures.ts
import { test as base, createBdd } from 'playwright-bdd';

export const test = base.extend({});
export const { Step: aistep } = createBdd(test);
```

### Create `steps.ts`

```typescript
// steps/steps.ts
import { runAgent, claudeCode } from 'openqa';
import { aistep } from './fixtures';

aistep(/^(.*)$/, async ({ page }, action: string) => {
  await runAgent(claudeCode('claude-haiku-4-5'), action, page, { verbose: true });
});
```

### Configure `playwright.config.ts`

```typescript
import { defineConfig } from '@playwright/test';
import { defineBddConfig } from 'playwright-bdd';

const testDir = defineBddConfig({
  featuresRoot: '../features',       // root of your features directory
  features: '../features/**/*.feature',
  steps: 'steps/*.ts',
});

export default defineConfig({
  testDir,
  timeout: 240000,
  use: { headless: true },
});
```

## Write Feature Files

Use `*` (asterisk) for a natural, AI-friendly step syntax:

```gherkin
Feature: TodoMVC Automation

  Scenario: Add todo item
    * I navigate to "https://demo.playwright.dev/todomvc/"
    * I add a new todo item "Buy groceries"
    * I should see "Buy groceries" in the todo list

  Scenario: Filter todos
    * I navigate to "https://demo.playwright.dev/todomvc/"
    * I add three todo items: "Task 1", "Task 2", and "Task 3"
    * I mark the first todo as completed
    * I click the Active filter
    * I should see 2 active todos
```

Standard `Given`/`When`/`Then` also works identically.

## Run Tests

```bash
npm run bddgen && playwright test --headed
```
