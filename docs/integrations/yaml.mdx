# YAML Tests

The simplest way to define browser tests — just a list of natural language steps.

## Setup

```bash
npx openqa init
# Select: YAML
```

## Example

```yaml
name: TodoMVC Tests

tests:
  - name: Add todo item
    steps:
      - Navigate to "https://demo.playwright.dev/todomvc/"
      - Add a new todo item "Buy groceries"
      - Verify "Buy groceries" appears in the todo list

  - name: Complete and filter todos
    steps:
      - Navigate to "https://demo.playwright.dev/todomvc/"
      - Add three todo items "Task 1", "Task 2", and "Task 3"
      - Mark the first todo as completed
      - Click the Active filter
      - Verify 2 active todos are shown
```

## Schema

| Field | Required | Description |
|-------|----------|-------------|
| `name` | Yes | Test suite name |
| `tests` | Yes | Array of test cases |
| `tests[].name` | Yes | Individual test name |
| `tests[].steps` | Yes | Natural language steps |

## Run Tests

```bash
cd .openqa
npm test
```

## Writing Good Steps

Be specific — the more detail you give, the better the agent performs:

<Tabs>
  <Tab title="Good">
    ```yaml
    - Navigate to "https://example.com/login"
    - Enter "user@test.com" in the email field
    - Click the "Sign In" button
    - Verify "Dashboard" appears in the heading
    ```
  </Tab>
  <Tab title="Avoid">
    ```yaml
    - Go to login
    - Log in
    - Check dashboard
    ```
  </Tab>
</Tabs>
